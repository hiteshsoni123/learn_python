What is Python?

A programming language is a special way to communicate with a computer. Just like we use Hindi, English, or Gujarati to talk with people, we use Python, Java, C, etc. to talk with computers.

Procedural Oriented Programming (POP)
  • Program = Step-by-step instructions (functions)
  • Data and functions are separate
  • Example: C language
  • Think of it like a recipe card – just steps written down
Object-Oriented Programming (OOP)
  • Program = Objects (things) that have data + actions
  • Data and functions are together in one place
  • Example: C++, Java, Python
  • Think of it like a chef – knows both the recipe (data) and how to cook (actions)

List Comprehensions
  • What is: A short way to create lists in Python using a single line of code
  • Comes from: Comes from Python's loops and list data type
  • Why: It makes code shorter, faster, and more readable than using loops
[expression for item in iterable if condition]
expression → What you want to store in the list
item → Variable that takes values from the iterable (like numbers, string, list)
iterable → A sequence (like list, range, string)
condition (optional) → Filter (only include items if condition is true)
Simple Example
Normal Way (Without List Comprehension)
squares = []
for i in range(1, 6):
    squares.append(i * i)
Pythonic Way (With List Comprehension)
squares = [i * i for i in range(1, 6)]

Error Handling
  • What is: A way to manage errors in code without crashing the program.
  • Comes from: Comes from Python's try-except block.
  • Why: Prevents application from stopping due to small errors.
try:
    # code that might cause an error
except ExceptionType:
    # code to handle the error
Simple Example
Without Error Handling
# This will crash if user enters 0
num = int(input("Enter a number: "))
result = 100 / num
print(result)
With Error Handling
try:
    num = int(input("Enter a number: "))
    result = 100 / num
    print(result)
except ZeroDivisionError:
    print("Please don't enter zero!")
except ValueError:
    print("Please enter a valid number!")
except Exception as e:
    print(f"Some other error occurred: {e}")

Type Hints
  • What is: Way to tell Python and developers what type of value a variable or function expects.
  • Comes from: Introduced in Python 3.5 with the typing module.
  • Why: Improves readability, reduces bugs, helps tools like mypy check code.
def function_name(parameter: type) -> return_type:
    # function code
Simple Example
Without Type Hints
def greet(name):
    return f"Hello, {name}"

result = greet("Amit") # Works
result = greet(123) # Also works, but might cause error later
With Type Hints
def greet(name: str) -> str:
    return f"Hello, {name}"

result = greet("Amit") # Good
result = greet(123) # Your IDE will highlight this as a potential problem

Naming Conventions, Design Standards, Formatting Rules

Follow these to write clean, professional Python code that everyone can understand easily.

Naming Conventions (PEP 8)
  • Variables & Functions: Use snake_case (e.g., customer_name, calculate_total())
  • Classes: Use PascalCase (e.g., SalesInvoice, CustomReport)
  • Constants: Use UPPER_SNAKE_CASE (e.g., STATUS_ACTIVE = "Active")
Design Standards
  • DRY (Don't Repeat Yourself): If you are writing the same code twice, make it a function
  • KISS (Keep It Simple, Stupid): Write simple, straightforward code. Avoid being too clever
  • Functions should do one thing: A function should have a single, clear purpose
Formatting Rules (PEP 8)
  • Indentation: Use 4 spaces per indentation level. Never use tabs
  • Line Length: Try to limit lines to a maximum of 79 characters
  • Spaces: Use spaces around operators (=, +, -) and after commas
  • Use a linter: Use tools like Flake8 or black (auto-formatter) in your VS Code to automatically check your style
